ML Implementation Core

A deep dive into the algorithmic backbone of SafeSurf v3.1, detailing our multi-model approach to zero-day phishing detection.

Model Orchestrator

Unified VotingClassifier Ensemble

Instead of relying on a single algorithm, SafeSurf employs a Soft-Voting Ensemble. This meta-classifier aggregates the weighted probabilities from five distinct structural experts, ensuring that the final verdict is robust against individual model biases and "blind spots."

📈

Calibrated XGBoost

The Heavy Lifter

Extreme Gradient Boosting that builds decision trees sequentially to correct previous errors. Integrated with Isotonic Regression for precise probability mapping.

Primary engine for complex structural pattern matching.

🌲

Random Forest

Robustness Specialist

An ensemble of independent decision trees that averages results to decrease variance and prevent overfitting on training data noise.

Provides baseline stability and handles high-dimensional noise.

⚡

HistGradientBoosting

Efficiency Specialist

Optimized for large-scale datasets, it bins continuous features into discrete intervals, making it incredibly fast for live inference loops.

Handles the massive PhiUSIIL 235k-URL data ingestion speed.

📐

LinearSVC

Boundary Specialist

Determines the optimal hyperplane (boundary) that separates Phishing and Safe classes with the maximum possible safety margin.

Ensures strict classification for ambiguous "edge case" URLs.

🔗

Logistic Regression

Linear Specialist

Predicts outcomes by analyzing the linear probability relationships between input features like URL length and hyphen frequency.

Analyzes basic, direct signals with high mathematical accuracy.

✍️

Bag-of-Words NLP

Lexical Fallback

A secondary Logistic Regression model that treats URLs as text tokens, specifically hunting for brand keywords and social engineering cues.

Acts as the 35% weighted "text-aware" sanity check.

The Hybrid Fusion Flow

SafeSurf splits analysis into two parallel paths. The Structural path (65%) interrogates the infrastructure while the Lexical path (35%) reads the text tokens. The final convergence happens in our soft-voting math combiner, providing a definitive risk verdict.

Implementation Note: Our migration from Stacking to VotingClassifier resolved Scikit-learn 1.6 / XGBoost 2.x attribute conflicts while preserving ensemble integrity.
